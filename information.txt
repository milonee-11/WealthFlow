🗄️ DATABASE DESIGN (MongoDB):
Collections:
├── Users
│   ├── Id: ObjectId (Primary Key)
│   ├── FullName: string
│   ├── Email: string (Unique)
│   ├── Password: string (Hashed)
│   ├── MonthlyBudget: double (Default: 2000)
│   ├── EmailNotifications: boolean (Default: true)
│   └── CreatedAt: DateTime
├── Expenses
│   ├── Id: ObjectId (Primary Key)
│   ├── UserId: string (Foreign Key to Users)
│   ├── Title: string
│   ├── Amount: double
│   ├── Category: string
│   ├── Note: string (Optional)
│   └── Date: DateTime
└── Alerts
    ├── Id: ObjectId (Primary Key)
    ├── UserId: string (Foreign Key to Users)
    ├── Type: string ('info', 'warning', 'danger')
    ├── Message: string
    ├── Severity: string
    ├── EmailSent: boolean (Default: false)
    └── Timestamp: DateTime

🔐 AUTHENTICATION FLOW:
1. User Registration (POST /api/auth/signup)
   ├── Input: { fullName, email, password }
   ├── Password: Hashed using SHA256
   ├── User: Created in MongoDB with EmailNotifications = true
   ├── Response: JWT Token + User Data
   └── Token: Stored in localStorage

2. User Login (POST /api/auth/login)
   ├── Input: { email, password }
   ├── Validation: Hash comparison with stored password
   ├── Response: JWT Token + User Data
   └── Authorization: Bearer token for protected routes

3. JWT Token Structure:
   ├── NameIdentifier: User Id
   ├── Email: User Email
   ├── Name: User Full Name
   └── Expiry: 7 days

💰 EXPENSE CRUD OPERATIONS (ExpensesController.cs):
1. Create Expense (POST /api/expenses)
   ├── Authentication: [Authorize] required
   ├── Input: { title, amount, category, note, date }
   ├── UserId: Extracted from JWT token
   ├── Process:
   │   ├── Save expense to MongoDB
   │   ├── Check budget thresholds
   │   └── Send automatic alerts/emails
   └── Response: Created expense object

2. Read Expenses (GET /api/expenses)
   ├── Authentication: [Authorize] required
   ├── UserId: Extracted from JWT token
   ├── Filter: expenses.Where(e => e.UserId == userId)
   └── Response: Array of user expenses

3. Update Expense (PUT /api/expenses/{id})
   ├── Authentication: [Authorize] required
   ├── Validation: Ensure expense belongs to user
   ├── Process: Update expense + budget check
   └── Response: Updated expense object

4. Delete Expense (DELETE /api/expenses/{id})
   ├── Authentication: [Authorize] required
   ├── Validation: Ensure expense belongs to user
   ├── Process: Delete expense from MongoDB
   └── Response: 200 OK

📧 BUDGET ALERT SYSTEM:
Automatic Budget Checking (CheckBudgetAndAlert method):
1. Trigger: After every expense add/update/delete
2. Process:
   ├── Fetch user's monthly budget
   ├── Calculate total expenses for current month
   ├── Calculate percentage: (total / budget) * 100
   ├── Determine alert level:
   │   ├── ≥75%: Info alert (blue)
   │   ├── ≥90%: Warning alert (yellow)
   │   ├── ≥100%: Danger alert (red)
   ├── Create alert record in MongoDB
   └── Send email if EmailNotifications = true

Email Service (EmailService.cs):
• SMTP Configuration for email sending
• Templates for budget alert emails
• Error handling for failed email delivery
• Supports both automatic and manual email triggers

Email Control Flow:
├── Frontend Toggle → EmailSettingsContext
├── API Call → PUT /api/auth/update-email-notifications
├── Backend → Update User.EmailNotifications field
└── Future Emails → Respect EmailNotifications flag
